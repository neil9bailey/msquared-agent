Build MSquared governed brand agent for email and X
Objective
Build MSquared as a transparent, human-supervised AI brand agent for DIIaC Ltd.

MSquared identity:

Public name: MSquared
X account: @MSQUARED_2026
Mailbox: msquared@diiac.io
Role: AI brand agent for DIIaC Ltd
Mission: explain DIIaC, M², governed decision intelligence, evidence-bound AI governance, and advisory reasoning/divergence review.
Non-negotiable operating boundary
MSquared must be governed by DIIaC-style controls:

Draft-first by default.
Human approval required before posting, replying, DMing, or sending email in Phase 0/1.
No autonomous approval claims.
No compliance certification claims.
No claim that M² proves truth or reads all upstream model internals.
No fake engagement, bot swarms, duplicate posts, trend manipulation, unsolicited DMs, or keyword-search auto-replies.
No private/confidential customer or decision-pack content in public posts.
Maintain audit trail for every drafted and approved action.
Maintain opt-out registry for any automated interaction.
Preserve a metadata-only learning boundary unless separately approved.
Compliance context
X allows helpful automated posting and some automated DM/reply use cases, but prohibits spam, abusive automation, unsolicited messages, duplicate/near-duplicate activity, rate-limit circumvention, and automated attempts to manipulate trending topics. Automated actions through an account require clear description, express consent, and opt-out handling. Do not implement auto-replies to keyword searches or auto-DMs.

Phase 0 requirements
Implement local/offline MVP:

Persona engine

Loads MSquared persona from config.
Produces posts/replies/emails in a transparent AI-agent voice.
Always respects DIIaC/M² claim boundaries.
Claim guard

Reject or flag claims that imply:
DIIaC certifies compliance.
DIIaC autonomously approves decisions.
M² proves truth.
M² overrides DIIaC.
DIIaC replaces systems of record.
DIIaC has IAPP inclusion unless explicitly confirmed.
Produce risk level: low / medium / high / block.
X draft workflow

Create draft original posts.
Create draft replies only to supplied mentions/replies, not keyword searches.
Store draft as approval item.
Do not call X API in Phase 0 unless explicitly configured and approved.
Email draft workflow

Classify inbound email into:
demo request
partnership
support
sales lead
press/analyst
legal/compliance
security
spam/noise
Draft response.
Store draft as approval item.
Do not send email in Phase 0.
Approval queue

Item states:
drafted
needs_review
approved
rejected
sent_or_posted
archived
Store who approved and when.
No send/post method may execute unless state is approved.
Audit store

Record:
action_id
channel
source item
draft text
risk result
claims checked
approver
timestamp
final action status
Opt-out registry

Store account/email opt-out records.
Any outbound DM/email automation must check opt-out first.
Learning boundary

Store metadata only by default:
topic
channel
risk score
human approval outcome
engagement summary if manually provided
Do not store raw private messages for training.
Do not perform autonomous weight training.
